# Client - Server Architecture

Server:

  • always on host
  • permanent IP address

Client:

  • communicate with server only, not with each other
  • may be intermittently connected and have dynamic IP addresses

# P2P Architecture

  • No always-on server
  • End systems communicate directly
  • Self scalability

What an Application Layer protocol defines:

  • types of messages - request, response, etc.
  • message syntax - fields in messages.
  • message semantics - meaning of data in fields.
  • rules - for when/how processes send & respond to messages.

# Web & HTTP

HTTP

  • Hypertext Transfer Protocol
  • the web's application layer protocol.
  • client/server model
  • stateless
  • uses TCP
Non-persistent HTTPPersistent HTTP
At most one object sent over TCP connection, then closed.Multiple objects can be sent over single TCP connection.

# HTTP Message Format

Request:

method URL version
header name: value
entity body

Response:

status line (protocol, status code, status phrase)

data

# HTTP Method Types

  • GET - request data
  • POST - send data
  • HEAD -
  • PUT - upload files
  • DELETE - delete by URL

# HTTP Response Codes

  • 200 - OK
  • 302 - Moved permanently
  • 400 - Bad request
  • 404 - Not found

Cookie

A packet of data sent by a server to a client which is returned every time the client subsequently accesses the sever.

# DNS Records

  • Type A

    • name: hostname
    • value: IP address

  • Type NS

    • name: domain
    • value: hostname of authoritative name

  • Type CNAME

    • name: alias for the real (canonical) name
    • value:canonical name

  • Type MX

    • value: mail sever associated with name

# Socket Programming

# TCP Process:

  1. Client contacts server first (server must be running).

  2. Server must have created socket to accept client.

  3. Client contacts by creating TCP socket with IP address & post number of server process.

  4. Server creates a new socket to communicate with that particular client.

    • server can talk to multiple clients
    • port numbers are used to identify clients
